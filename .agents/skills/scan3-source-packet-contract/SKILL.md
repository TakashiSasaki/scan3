# Name
scan3-source-packet-contract

# Status
Active

# Purpose
Synchronize source packet constraints across schema, expectation catalog, tests, constraint matrix, and accepted artifacts to prevent contract divergence.

# Preconditions
- Owner decisions regarding source packets must be available.
- Schema and matrix files must exist.

# Authoritative References
- policy/project.md
- reconstruction/source-packet.schema.json
- reconstruction/source-packet-constraints.json

# Inputs
- Modified or new owner decisions
- Modified or new schema constraints

# Procedure
1. decisionsとdecision requestsを読む。
2. schema dialectとowner decisionsを確認する。
3. schemaを更新する。
4. static-fixture outcome state machineを検証し、fixture expectation catalogを更新する。
5. exact schemaErrorPathsを更新する。
6. static, dynamic, および direct-test registriesと constraintIds を更新する。
7. exact Ajv evidence testsを実行する。
8. operational error-code evidence closureを実行する。
9. constraint matrixを再生成・検証する。
10. accepted-artifact dependency closureを更新する。
11. unresolved semantic changesをdecision gatesを通して分類する。
12. unresolved semantic issueがあればdecision requestを作る。

# Decision Gates
- Does the change require a new owner decision? If so, stop and request.

# Stop Conditions
- Contract validation script passes.
- All files are synchronized.

# Verification
Run `node scripts/validate-source-packet-contract.cjs`

# Outputs
- Updated schema
- Updated matrix
- Updated tests
- Synchronized contract state

# Non-Goals
- Executing source packet processing (this is only for the contract)
